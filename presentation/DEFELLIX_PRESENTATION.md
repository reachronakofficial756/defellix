# SLIDE 09 — THE BLOCKCHAIN ADVANTAGE

## Why blockchain? Because "I said so" isn't evidence.

### WHAT GETS STORED ON BASE L2

When a client signs a contract on Defellix, this is recorded **permanently** on the blockchain:

| Data Point | What It Proves |
|------------|----------------|
| **Contract hash** | Exact terms agreed upon — any change would break the hash |
| **Block timestamp** | Proves *when* both parties agreed |
| **Transaction hash** | Unique, permanent, publicly verifiable record |
| **Both wallet addresses** | Links freelancer AND client to the contract |
| **Milestone breakdown** | Payment terms cannot be disputed retroactively |

### THIS MAKES CHEATING MATHEMATICALLY IMPOSSIBLE.

> A client cannot say "we never agreed to that" when the agreement is permanently encoded in an immutable ledger.

**Network: Base L2 (Ethereum Layer 2)**
- Cheap: ~$0.01–$0.05 per contract write
- Fast: Seconds, not minutes
- Secure: Ethereum-grade security

# SLIDE 10 — DISPUTE RESOLUTION

## When things go wrong, you're protected. For free.

### SECTION 65B CERTIFICATES — YOUR LEGAL SHIELD

Under the **Indian Evidence Act, 1872, Section 65B**, digital records are admissible in court **only if** they meet specific certification requirements.

Defellix generates these certificates **automatically** from blockchain data.

**What the certificate includes:**

```
✓ Full contract terms (project name, amount, deadline)
✓ Client signature date and digital metadata
✓ Milestone breakdown with payment amounts
✓ Complete work submission timeline
✓ Blockchain transaction hash
✓ Block number and network proof
✓ Cryptographic proof of document integrity
```

**Cost: FREE**
**Time to generate: Seconds**
**Admissibility: Indian courts, under Section 65B**

### Without Defellix: 12.5 months average dispute resolution
### With Defellix: Walk into court with a certificate in hand.

# APPENDIX B — PLATFORM TECHNICAL DETAILS

## For due-diligence or developer audiences

### Architecture
- **4 microservices** (Go 1.24+): Auth (8080), User (8081), Contract (8082), Blockchain (8083)
- **Database:** PostgreSQL (shared `defellix` instance)
- **ORM:** GORM
- **Router:** Chi v5
- **Blockchain client:** go-ethereum v1.16.8

### Blockchain
- **Network:** Base L2 (Layer 2 on Ethereum)
- **Testnet:** Base Sepolia (Chain ID: 84532)
- **Mainnet:** Base Mainnet (Chain ID: 8453)
- **RPC:** Alchemy
- **Wallet:** Custodial, encrypted with AES-level key

### Security
- Private keys encrypted at rest, never exposed to frontend
- JWT authentication (access + refresh tokens)
- OAuth: Google, LinkedIn, GitHub
- bcrypt password hashing

### Key Endpoints
- `POST /api/v1/contracts` — Create contract
- `GET /api/v1/contracts/:id/public` — Public client view (no auth)
- `POST /api/v1/contracts/:id/sign` — Client signs (triggers blockchain write)
- `GET /api/v1/contracts/:id/certificate` — Generate Section 65B certificate
